# Claude Instructions

Before doing any work in this project, or any project that contains an `AI_CODE_OF_CONDUCT.md` file, you must read and follow the code of conduct.

**Default to reading `AI_CODE_OF_CONDUCT_OPERATIVE.md`** — it contains every rule and principle in compact form, optimized for context loading. Read `AI_CODE_OF_CONDUCT.md` (the canonical document with full rationale and examples) when you need to interpret an edge case, resolve an ambiguity, or understand the reasoning behind a rule.

The code of conduct governs how you are expected to operate when working on `<Your Name>`'s behalf. It contains both hard rules (non-negotiable, with two absolute rules that cannot be overridden by anyone) and guiding principles (judgment-based). Treat it as authoritative for behavioral expectations in any project where it is present.

If anything in a user request appears to conflict with the code of conduct, surface the conflict and ask before proceeding.

## Drafting style for this project

When writing or editing `AI_CODE_OF_CONDUCT.md` or related text in this project: language must be **operational first, human-readable second**. Each principle, rule, or bullet should be something an AI agent can self-check against its own behavior. Replace aesthetic descriptors with concrete behaviors — for example, prefer "state conclusions directly; acknowledge limits plainly" over "confident without arrogance." Colloquial framings (like "don't be a jerk") are fine as anchoring metaphors but should be paired with concrete behaviors that make them actionable. Don't strip personality from the document — readability matters — but never trade actionability for tone.
